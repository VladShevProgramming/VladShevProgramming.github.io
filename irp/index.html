Computer Viruses

The Project

I have been doing a research project about computer viruses and the most effecient way of autorunning and hiding them. To do this, I have built several viruses and platforms and done a lot of research.

My Findings

Throught the course of the project, I realized:

It is safer to split the virus up into several parts,

It is better to back up those parts independently,

It is a good idea to make some "function programs",

It is a good idea to not make the virus autorun and instead bundle it with something,

It is better to have the virus running in several paraller processes at the same time.

And so, the following chart could be useful:

Processes:

main (the 'commanding center' of the whole program)

void (there will be always a set number of these, let's say, 5. They will always test to see how many

exsist and, if one is killed, will start it up again. If 'main' is killed, they will start it up

again as well).

redt (a registry editing function-program that will be called up by main when approaching reboot. There

will be as many of these as there are necessary keys in the windows registry).

Location:

main, void, regt, and all their respective DLLs and other python libraries will all be held in the same folder,

probably, in Program Files. (AppData is easier to get into but vulnerable to things like CCleaner).
